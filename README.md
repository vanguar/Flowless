# Лендинг «Автоматизация бизнес-процессов»

Одностраничник с формой заявки. Светлая/тёмная тема и два языка (RU/UA)
переключаются кнопками в шапке и запоминаются в браузере.

## Файлы
- `index.html` — разметка
- `styles.css` — стили (темы через CSS-переменные)
- `script.js` — переводы, переключатели темы/языка, форма, анимации

## 1. Настройка (обязательно)
Откройте `script.js`, блок `CONFIG` в самом верху, и впишите свои данные:

```js
const CONFIG = {
  telegram: "username",          // ваш ник без @
  email: "you@example.com",      // ваша почта
  phone: "+70000000000",         // телефон
  formEndpoint: ""               // см. пункт 2
};
```

## 2. Форма заявки
GitHub Pages — статический хостинг, сервер форму не обработает. Варианты:

- **С сервисом (рекомендую):** заведите бесплатный ящик на
  [formspree.io](https://formspree.io), получите endpoint вида
  `https://formspree.io/f/xxxxxx` и вставьте его в `CONFIG.formEndpoint`.
  Заявки будут приходить на почту.
- **Без сервиса:** оставьте `formEndpoint: ""`. Тогда при отправке откроется
  почтовый клиент с уже заполненным письмом (fallback).

## 3. Деплой на GitHub Pages
1. Создайте репозиторий и загрузите в него `index.html`, `styles.css`, `script.js`
   (всё в корень).
2. **Settings → Pages → Build and deployment**: Source — *Deploy from a branch*,
   ветка `main`, папка `/ (root)`, **Save**.
3. Через минуту сайт будет доступен по адресу
   `https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПО/`.

Все пути относительные, поэтому работает и в подпапке репозитория.

## Правки
- Тексты переводов — объект `I18N` в `script.js` (ключи `ru` и `uk`).
- Цвета — переменные в начале `styles.css` (`:root` — светлая,
  `[data-theme="dark"]` — тёмная).
